## 🎯 演示目标

向政府和产业园区展示 Demand-OS 的核心能力：**通过对话式交互，智能引导工厂用户完成产品委托开发流程**。

---

## 🚀 快速启动

### 1. 启动开发服务器

```bash
cd /home/ubuntu/demand-os/web
npm run dev
```

### 2. 访问演示页面

打开浏览器访问: `http://localhost:3000/agent-chat`

## 📖 演示流程

### 第一步：展示欢迎界面

页面加载后，您会看到：
- **标题**: "工厂委托开发助手"
- **欢迎消息**: "您好！我是工厂委托开发助手。请告诉我您想开发的产品，我会帮您匹配合适的工厂。"
- **输入框**: 用于输入需求
- **发送按钮**: 绿色的发送图标

**演示话术**:
> "这是我们的工厂委托开发助手。它可以通过对话的方式，引导工厂用户完成产品委托开发的整个流程。让我们来看一个例子。"

### 第二步：输入需求

在输入框中输入：
```
我想开发一款智能蓝牙音箱
```

然后点击发送按钮。

**演示话术**:
> "假设一个工厂想要开发一款智能蓝牙音箱，他只需要在这里输入他的需求。"

### 第三步：展示执行计划

发送后，系统会立即响应：
- **用户消息**: "我想开发一款智能蓝牙音箱"（显示在右侧，绿色背景）
- **系统响应**: "好的，我已经为您创建了执行计划。让我们开始吧！"（显示在左侧，深色背景）
- **执行计划卡片**: 在右侧显示，包含 5 个步骤

**演示话术**:
> "系统立即理解了用户的意图，并自动生成了一个完整的执行计划。您可以看到右侧的执行计划卡片，包含了 5 个步骤：
> 1. 收集产品信息 - 进行中
> 2. 市场分析 - 等待中
> 3. 定义合作策略 - 等待中
> 4. 工厂资质审核 - 等待中
> 5. 提交申请 - 等待中
> 
> 第一个步骤已经开始执行，状态显示为'进行中'。"

### 第四步：解释核心优势

**演示话术**:
> "这个系统的核心优势在于：
> 1. **智能理解**: AI 自动理解用户的意图，无需用户学习复杂的操作
> 2. **逐步引导**: 系统会一步步引导用户完成所有必要的信息收集
> 3. **实时反馈**: 用户可以实时看到每一步的执行状态
> 4. **简约科技感**: 界面简洁优雅，符合现代化的设计标准
> 
> 接下来，系统会在聊天界面中嵌入一个表单，引导用户填写产品的详细信息。填写完成后，AI 会自动进行市场分析，并逐步完成后续的所有步骤。"

## 🎨 核心亮点

### 1. 对话式交互
用户只需要像聊天一样输入需求，系统就能理解并自动执行。

### 2. 逐步引导式
系统会在每一步提示用户需要做什么，用户只需按照提示操作。

### 3. 实时进度展示
右侧的执行计划卡片实时展示任务进度，用户可以清楚地知道当前在第几步。

### 4. 简约的科技感
界面设计简洁优雅，深色主题，符合政府和产业园区的专业形象。

## 📊 技术架构（可选，用于技术性演示）

如果观众对技术细节感兴趣，可以补充说明：

### 核心组件
1. **Intent Parser**: 使用 LLM 理解用户意图
2. **Task Executor**: 状态机驱动的任务执行引擎
3. **Factory ODM Agent**: 预定义的工厂委托开发流程
4. **Execution Plan Card**: 实时展示执行计划和进度

### 技术栈
- **前端**: Next.js + React + TypeScript + TailwindCSS
- **后端**: Next.js API Routes
- **AI**: OpenAI GPT-4
- **存储**: 内存存储（生产环境可升级为 Redis）
