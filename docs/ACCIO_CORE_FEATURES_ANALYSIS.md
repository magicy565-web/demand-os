## 1. 核心交互流程

### 用户输入 → AI 规划 → 自动执行 → 结果呈现

Accio 的核心流程可以总结为以下四个阶段：

1. **用户输入单一指令**
   - 用户通过文字或语音输入一个复杂的需求
   - 例如："帮我分析这个产品的市场趋势，找到合适的供应商，并生成询盘邮件"

2. **AI 自动规划多步骤执行计划**
   - 系统将用户的指令分解成多个子任务
   - 展示给用户一个清晰的执行计划（如：趋势分析 → 产品研究 → 供应商搜索 → 生成策略）

3. **AI 自动执行所有子任务**
   - 系统自动执行每个子任务，无需用户干预
   - 实时展示执行进度（如："正在进行供应商搜索..."）

4. **多种形式呈现结果**
   - 综合研究报告（带图表）
   - 供应商列表（可选择和比较）
   - 询盘邮件草稿
   - 手机推送通知

## 2. 核心特性详解

### 2.1 智能意图理解

**关键点**：用户只需输入一句话，AI 就能理解复杂的多步骤需求。

**示例**：
- 用户输入："帮我分析这个 TikTok 视频的爆款潜力，并找到合适的工厂"
- AI 理解为：
  1. 分析视频内容
  2. 评估市场潜力
  3. 提取产品信息
  4. 匹配工厂
  5. 生成报告

**技术实现**：
- 使用 LLM（如 GPT-4）解析用户输入
- 将用户意图映射到预定义的任务模板
- 动态生成执行计划

### 2.2 动态步骤生成

**关键点**：系统会将用户的指令分解成一个多步骤的执行计划，并展示给用户。

**示例**：
```
用户输入: "帮我分析这个产品的市场趋势"

AI 生成的执行计划:
1. 趋势分析 (Trend Analysis)
2. 产品研究 (Product Research)
3. 生成策略 (Generate Strategy)
```

**UI 呈现**：
- 在对话界面中展示一个"执行计划"卡片
- 每个步骤有一个图标和简短描述
- 用户可以看到整个流程的全貌

### 2.3 实时进度展示

**关键点**：在执行过程中，用户能看到每一步的进展。

**示例**：
```
✓ 趋势分析 - 已完成
⏳ 产品研究 - 进行中...
⏸ 生成策略 - 等待中
```

**UI 呈现**：
- 每个步骤有一个状态指示器（✓ 完成、⏳ 进行中、⏸ 等待中）
- 当前正在执行的步骤会高亮显示
- 可以展开查看每一步的详细日志

### 2.4 结构化结果呈现

**关键点**：结果不是纯文本，而是结构化的、可交互的。

**示例**：

#### 综合研究报告
- 包含图表（如市场趋势图、价格对比图）
- 分段展示（如：市场分析、产品特性、竞争对手）
- 支持导出为 PDF

#### 供应商列表
- 卡片式布局，每个供应商一个卡片
- 包含关键信息（名称、位置、评分、价格）
- 支持筛选和排序
- 可以直接点击"联系供应商"

#### 询盘邮件草稿
- 预填充的邮件模板
- 用户可以编辑后直接发送

## 3. UI/UX 设计要点

### 3.1 视觉风格

- **深色模式**：主背景色为深色（如 `#0a0a0f`）
- **霓虹绿色强调**：使用醒目的绿色（如 `#00FF00`）作为主强调色
- **现代简洁**：去除不必要的装饰，突出内容

### 3.2 对话界面

- **聊天式布局**：类似 ChatGPT 的对话界面
- **输入框**：支持文字和语音输入
- **消息气泡**：用户消息和系统消息使用不同颜色

### 3.3 执行计划卡片

- **卡片式设计**：执行计划以卡片形式展示
- **步骤列表**：每个步骤有图标、标题和状态指示器
- **可折叠**：用户可以展开查看详细日志

### 3.4 结果呈现

- **多种形式**：根据结果类型选择合适的呈现方式
  - 文本 → 段落
  - 数据 → 表格或图表
  - 列表 → 卡片式布局
- **可交互**：用户可以点击、筛选、排序

## 4. 技术架构要点

### 4.1 后端架构

```
用户输入
    ↓
意图理解 (LLM)
    ↓
任务规划 (Task Planner)
    ↓
任务执行引擎 (Task Executor)
    ↓
结果聚合 (Result Aggregator)
    ↓
结果呈现
```

### 4.2 关键组件

1. **意图理解器 (Intent Parser)**
   - 使用 LLM 解析用户输入
   - 提取关键信息（如产品名称、目标市场）
   - 映射到任务模板

2. **任务规划器 (Task Planner)**
   - 根据用户意图选择合适的任务模板
   - 动态生成执行计划
   - 返回步骤列表

3. **任务执行引擎 (Task Executor)**
   - 按顺序执行每个步骤
   - 实时更新进度
   - 处理错误和重试

4. **结果聚合器 (Result Aggregator)**
   - 收集所有步骤的结果
   - 生成结构化的最终结果
   - 支持多种格式（JSON、Markdown、HTML）

## 7. 总结

Accio 的核心优势在于**"一句话完成复杂任务"**。用户无需了解背后的流程，只需描述需求，AI 就会自动规划、执行并呈现结果。

要真正对标 Accio，我们需要实现：
1. **智能意图理解**：用 LLM 解析用户输入
2. **动态步骤生成**：AI 自动生成执行计划
3. **实时进度展示**：每个步骤的实时状态
4. **结构化结果呈现**：卡片、图表、表格等多种形式

当前的实现只完成了约 10%，主要是基础的对话框架。接下来需要重点实现上述四个核心特性。
