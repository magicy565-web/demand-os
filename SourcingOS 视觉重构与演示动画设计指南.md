# SourcingOS 视觉重构与演示动画设计指南

**作者**: Manus AI  
**日期**: 2026年2月5日

## 1. 引言

本指南旨在为 SourcingOS 平台的前端界面提供一套全面的视觉重构与演示动画设计策略。我们将专注于提升用户体验，通过优化色彩、排版、组件样式和交互动效，将现有“初级交付”的 UI 提升至“产品级”的专业水准。同时，本指南将提供详细的 GIF 演示动画脚本，指导您如何制作出具有冲击力、能有效传达产品核心价值的演示内容。本指南将完全规避技术实现细节，侧重于设计美学和用户感知。

## 2. 项目核心视觉理念

SourcingOS 的核心理念是“以数换单，以单强产”和“跨境产业带数字化链主平台”。因此，视觉设计应传达以下关键信息：

*   **专业与信任**：作为“链主”平台，需要展现出权威性、可靠性和数据驱动的专业感。
*   **效率与智能**：通过简洁的布局、流畅的动效和清晰的信息流，体现 AI 带来的高效与便捷。
*   **现代与科技**：采用现代设计趋势，如微渐变、柔和阴影、清晰的几何线条，营造科技感。
*   **活力与创新**：在专业基调中融入适度的活力色彩，展现创新精神。

## 3. 品牌视觉识别 (Visual Identity) 指南

### 3.1 色彩方案

基于您提供的设计稿，我们提炼出以下核心色彩，并建议其在 UI 中的应用场景：

| 色彩名称 | 色值 (HEX) | 应用场景 | 情感联想 |
| :------- | :--------- | :------- | :------- |
| **主品牌蓝** | `#3B82F6`  | 核心按钮、强调元素、链接、进度条 | 科技、专业、信任、效率 |
| **辅助绿** | `#10B981`  | 成功状态、收益增长、积极反馈 | 生长、活力、成功、环保 |
| **警示橙** | `#F59E0B`  | 警告、待处理、中性提示 | 注意、活力、创新 |
| **警示红** | `#EF4444`  | 错误、超出预算、风险提示 | 警示、停止、重要 |
| **背景灰** | `#F8FAFC`  | 页面背景、卡片背景（浅色模式） | 简洁、现代、留白 |
| **文字深灰** | `#1F2937`  | 主要文本、标题 | 稳重、清晰、专业 |
| **文字浅灰** | `#6B7280`  | 次要文本、描述、提示信息 | 柔和、辅助、易读 |

### 3.2 字体排版

*   **主标题/强调文字**：建议使用现代无衬线字体，如 `Inter`, `Roboto`, `PingFang SC` (中文)，字重适中，突出力量感和科技感。
*   **正文/辅助文字**：使用易读性高的无衬线字体，如 `Inter`, `Noto Sans SC` (中文)，字号不宜过小，保持良好的行高和字间距。
*   **数据/代码**：使用等宽字体，如 `Fira Code`, `JetBrains Mono`，以增强数据和代码的专业感。

**Claude 设计提示词示例**：
```text
请为 SourcingOS 平台推荐一套字体组合，包括主标题、正文和数据展示字体。要求兼顾中英文显示效果，并提供相应的 Tailwind CSS 字体配置建议。
```

### 3.3 图标与视觉元素

*   **图标风格**：建议使用简洁、线条化的图标，与整体现代科技感保持一致。可参考 `Lucide Icons` 或 `Feather Icons`。
*   **背景纹理**：沿用您设计稿中的 CAD 网格背景，可考虑增加微弱的动态效果或渐变叠加，提升科技感。
*   **阴影与边框**：使用柔和的阴影 (`shadow-md`, `shadow-lg`) 和细致的边框，增加元素的层次感和立体感。

## 4. Module 01: 需求上传 (Hero Section & Demand Form) 视觉重构

### 4.1 Hero Section 优化建议

*   **“以数换单，以单强产”标题**：
    *   **视觉冲击力**：增大字号，利用 `text-balance` 优化断行，使用渐变色或描边效果突出“以单强产”部分。
    *   **动画**：标题文字可以有更具力量感的入场动画（如从中心扩散或逐字显现）。
*   **输入框与按钮**：
    *   **一体化设计**：将搜索框和“立即匹配产能”按钮设计成一个视觉整体，按钮可略微突出。
    *   **输入提示**：打字机效果的文本应更流畅，光标闪烁更自然。
    *   **按钮动效**：鼠标悬停时，按钮应有明显的反馈（如背景色加深、轻微上浮）。
*   **流程图 (Process Flow)**：
    *   **视觉清晰**：步骤图标和文字应清晰易读，连接线可增加微弱的动画效果（如流动感）。
    *   **色彩区分**：每个步骤的图标或背景色可略有不同，增加视觉趣味性。
*   **数据统计 (Data Stats)**：
    *   **简洁有力**：数字应足够大且醒目，单位和描述文字保持简洁。
    *   **微动效**：数字滚动动画或加载动画，增强数据带来的冲击力。

**Claude 设计提示词示例**：
```text
请为 SourcingOS 的 Hero Section 重新设计“以数换单，以单强产”标题的样式，使其更具视觉冲击力。考虑使用 Tailwind CSS 的渐变色、`text-shadow` 或 `text-stroke` 效果。同时，为“立即匹配产能”按钮设计一个更吸引人的 hover 动画。
```

### 4.2 Demand Form (Dialog) 优化建议

*   **对话框样式**：
    *   **背景模糊**：对话框弹出时，背景可增加轻微的模糊效果，突出表单。
    *   **圆角与阴影**：使用更柔和的圆角和深度适中的阴影，提升质感。
*   **表单布局**：
    *   **分组**：将相关字段进行逻辑分组，使用分隔线或卡片区分。
    *   **标签与提示**：标签文字清晰，输入框内的 `placeholder` 提供具体示例。可考虑在字段下方添加简短的辅助说明文字。
*   **输入框与文本域**：
    *   **焦点效果**：当输入框获得焦点时，边框应有明显的颜色变化或光晕效果。
    *   **错误提示**：错误信息应以红色文字或图标在相关字段下方清晰显示，避免遮挡。
*   **提交按钮**：
    *   **加载状态**：加载图标应平滑旋转，按钮文字变为“提交中...”。
    *   **成功/失败反馈**：提交成功后，按钮可短暂变为绿色并显示“提交成功”，失败则变为红色并显示“提交失败”。

**Claude 设计提示词示例**：
```text
请为 SourcingOS 的需求提交表单（DemandForm）设计一套更现代、更专业的样式。重点优化输入框的焦点效果、错误提示的视觉呈现，以及提交按钮的加载和反馈动画。使用 Tailwind CSS 和 Shadcn UI 的组件，并提供具体的 CSS 类名建议。
```

## 5. Module 02: AI 拆单 (C2M Engine) 视觉重构

### 5.1 整体布局与信息层级

*   **左右分栏**：保持左侧配置选项、右侧配置摘要的布局。确保两栏宽度比例协调，内容对齐。
*   **卡片设计**：每个配置选项（主材、市场、面料）都应封装在独立的卡片中，卡片之间留有足够的间距，提升可读性。
*   **标题与描述**：每个模块和子模块的标题应清晰，描述文字简洁明了。

### 5.2 配置选项优化

*   **选中状态**：
    *   当用户选择某个物料、市场或面料时，对应的卡片应有**明显的视觉反馈**：例如，边框变为品牌蓝，背景色略微加深，并增加一个轻微的阴影效果。
    *   可考虑在选中卡片右上角添加一个小的“√”图标，进一步明确选中状态。
*   **价格系数显示**：
    *   **色彩区分**：正向价格系数（加价）使用警示橙或红色，负向价格系数（减价）使用辅助绿。
    *   **动效**：当价格系数变化时，数字可以有微小的动画效果（如数字滚动或淡入淡出）。
*   **滑块 (Slider)**：
    *   **自定义样式**：滑块的轨道和滑块本身应与品牌色调保持一致，滑块可设计成圆形或方形，带有微弱的阴影。
    *   **实时数值提示**：当用户拖动滑块时，应在滑块上方或旁边实时显示当前数值（如“¥500K”、“100 件”），提升交互体验。

**Claude 设计提示词示例**：
```text
请为 SourcingOS 的 AI 拆单页面（Module 02）设计一套更具交互感的样式。重点优化物料选择卡片的选中状态视觉反馈，包括边框、背景色和阴影。同时，为价格系数的显示设计一个更直观的色彩方案，并为滑块添加自定义样式和实时数值提示。
```

### 5.3 配置摘要优化

*   **预估总价**：
    *   **突出显示**：使用更大的字号、更粗的字重和主品牌蓝，使其成为视觉焦点。
    *   **动态更新**：当配置变化时，总价数字应有平滑的滚动更新动画。
*   **预算符合性**：
    *   **醒目提示**：使用大尺寸的 `CheckCircle2` (绿色) 或 `AlertTriangle` (红色) 图标，配合相应的背景色和文字，快速传达信息。
    *   **差额显示**：清晰显示“差额预算”或“超出”的具体金额。
*   **匹配供应商列表**：
    *   **简洁呈现**：列表应简洁，只显示关键信息（如供应商名称和交期）。
    *   **加载动效**：如果供应商列表是异步加载的，可使用骨架屏或加载动画。
*   **“确认配置”按钮**：
    *   **视觉层级**：作为页面的主要操作，应具有高视觉层级，使用主品牌蓝，并带有右箭头图标。
    *   **动效**：鼠标悬停时，按钮可有轻微的缩放或背景色变化。

**Claude 设计提示词示例**：
```text
请为 SourcingOS 的 AI 拆单页面右侧的“配置摘要”区域设计一套更具数据可视化效果的样式。重点突出“预估总价”和“预算符合性”的视觉呈现。考虑为总价添加数字滚动动画，并为预算符合性提示设计更醒目的图标和色彩组合。提供 Tailwind CSS 和 Shadcn UI 的实现建议。
```

## 6. GIF 演示动画设计脚本

高质量的演示动画能直观地展示产品功能和价值。以下是为 SourcingOS 核心模块设计的 GIF 演示脚本，旨在突出其 AI 驱动的智能和高效。

### 6.1 通用原则

*   **流畅性**：动画帧率高，过渡自然。
*   **聚焦**：突出关键操作和信息变化，可使用局部放大或高亮。
*   **简洁**：每个 GIF 专注于一个核心功能，时长不宜过长（建议 10-20 秒）。
*   **高质量**：录制分辨率高，无水印。
*   **文字说明**：GIF 旁可配简短文字说明，解释操作和效果。

### 6.2 Module 01: 需求上传 GIF 脚本

**目标**：展示用户如何快速提交场景化采购需求，并由 AI 自动解析。

1.  **场景 1 (0-3s)**：SourcingOS 首页 Hero Section 动画，标题“以数换单，以单强产”和打字机效果的输入框（“200间客房精品酒店，东南亚风格，预算$800K...”）流畅展示。
2.  **场景 2 (3-7s)**：鼠标悬停在“立即匹配产能”按钮上，按钮有微动效。点击按钮，需求提交表单以平滑的动画从屏幕中央弹出，背景略微模糊。
3.  **场景 3 (7-15s)**：表单自动填充（快速演示），用户点击“立即匹配产能”按钮。按钮显示加载动画（如旋转图标），然后短暂显示“提交成功”。
4.  **场景 4 (15-20s)**：页面平滑过渡到 Module 02 (AI 拆单) 页面，展示加载状态。

### 6.3 Module 02: AI 拆单 (C2M Engine) GIF 脚本

**目标**：展示 C2M 引擎的柔性定制能力，实时价格计算和预算控制。

1.  **场景 1 (0-3s)**：Module 02 页面加载完成，显示初始配置（如默认主材、面料、市场），右侧配置摘要显示初始预估总价和预算符合性。
2.  **场景 2 (3-8s)**：用户在左侧“主材选型”区域，点击选择不同的主材（如从“北美白橡木”切换到“美国黑胡桃木”）。被选中的卡片有明显视觉反馈，右侧“预估总价”和“价格系数”实时滚动更新。
3.  **场景 3 (8-13s)**：用户在“软装面料”区域，点击选择不同的面料。同样，选中卡片有视觉反馈，右侧总价实时更新。
4.  **场景 4 (13-17s)**：用户拖动“预算范围”滑块，滑块上方实时显示当前预算值。当总价超出预算时，“符合预算”提示变为“超出预算”，并以红色醒目显示。
5.  **场景 5 (17-20s)**：用户拖动“起订量 (MOQ)”滑块，右侧“匹配工厂”数量实时更新。最后，鼠标悬停在“确认配置，进入拼柜环节”按钮上，按钮有微动效。
